🌟 1. 什么是 Event Bus？

Vert.x Event Bus 是一个轻量级的消息总线，用于 Verticle 之间乃至应用之间的通信。它具有以下特性：

  • 进程内通信（Local）
  • 跨 Verticle 通信
  • 跨 JVM 通信（Cluster 模式）
  • 浏览器通信（SockJS/WebSocket Bridge）

也就是说：

Event Bus 不只是内部通信总线，它是一种天然支持分布式的消息机制。

🎯 2. Event Bus 的消息模式（非常重要）

Event Bus 提供三种基础通信模式，理解它们是使用 Event Bus 的关键。

① Send（点对点，Request/Response 模式）

只会发送给一个消费者。

消费者：

适用场景：

  • RPC 风格调用
  • 请求–响应业务处理链
  • 需要返回值的场景

② Publish（发布/订阅）

发送到所有订阅者（广播）。

适用：

  • 状态同步
  • 系统通知
  • 广播事件

③ Send with reply（请求–响应）

Send 就是点对点，带上 reply() 就是同步调用风格。

多数业务场景用它。

🧠 9. Event Bus 的线程安全原则

重点原则：每个 Verticle 总是在同一个 Event Loop 上运行。

因此：

  • 不需要 synchronized
  • 不需要锁
  • 数据不需要额外保护
  • 类似 Actor 模型，自然线程安全

Event Bus + Verticle 构成了 Vert.x 的“无锁”架构核心。

🎯 11. Event Bus 使用最佳实践

推荐做法 原因
用 request/reply 处理业务逻辑 更清晰的 API 调用风格
事件头传递 traceId 适合链路追踪
Service Verticle 最好分组组织 结构清晰，可扩容
避免大对象消息 Event Bus 适合小消息、快速传输
分布式 Event Bus 不要传超大 payload 节点间网络传输会变慢
